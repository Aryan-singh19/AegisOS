# Contributing to AegisOS

## Workflow

1. Fork and create a feature branch.
2. Open an issue if the change is large.
3. Submit a PR with tests/docs updates.
4. Ensure CI checks pass before requesting review.

## Commit Style

- Use concise, imperative commit messages.
- Keep one logical change per commit.

## Pull Request Expectations

- Describe the problem and approach clearly.
- Include risk notes for kernel/security changes.
- Add validation steps and expected outcomes.

## Priority Workstreams

- Kernel primitives (process, memory, IPC).
- Sandboxing and capability system.
- Build and packaging automation.
- Compatibility layer architecture.
